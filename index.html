Управление ветками. Работаем с репозиторием из задачи 1. 1. Создать и перейти в новую ветку my-first-branch из основной ветки main , используем команду git checkout -b my-first-branch 2. Создать файл style.css на одном уровне с index.html 3. Добавить в этом файле любые стили для заголовка h1 4. Подключить style.css на страницу и проверить, чтобы стили применились 5. Проиндексировать и сделать коммит изменений 6. Слить ветку my-first-branch в ветку main командой git merge my- first-branch , предварительно перейдя в ветку main 7. Отправить обновленную ветку main на удаленный репозиторий

Делаем первый pull request. Работаем с репозиторием из задачи 1. 1. Создать и перейти в новую ветку my-second-branch из основной ветки main 2. Добавить на страницу параграф с произвольным текстом 3. Задать стили для параграфа (цвет текста, размер шрифта, отступы)

4. Проиндексировать и сделать коммит изменений 5. Отправить текущую ветку на удаленный репозиторий командой git push origin my-second-branch 6. Перейти в свой репозиторий на github, затем во вкладку Pull requests 7. Нажимаем зеленую кнопку New pull request 8. Выбираем в base ветку main , в compare — my-second-branch 9. Нажимаем кнопку Create pull request 10. Просматриваем все изменения, если все верно принимаем pull request, для этого надо нажать на кнопку Merge pull request, затем Confirm merge 11. На удаленном репозитории рабочая ветка слилась в основную, соответственно my-second-branch теперь можно удалить через кнопку Delete branch 12. Возвращаемся в локальный репозиторий в ветку main , она сейчас отличается от одноименной ветки на github, поэтому для синхронизации изменений, запускаем команду git pull
